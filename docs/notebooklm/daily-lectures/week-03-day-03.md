## 2. Mini lecture
The week focus is A+ Core 2: operating systems, installs, file systems, and admin basics, and today's objective is apply the concepts with examples, drills, or comparisons. The main terms in scope are Upgrade Path and Command Line. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Upgrade Path is supported route for moving a system from one version, edition, or platform state to another. It reduces deployment risk by identifying which transitions can be performed without a full rebuild. Mechanically, it checks the current state, validates supported transitions, and then applies the new version while preserving or migrating supported data. Command Line is text-based interface for executing operating system commands. It accepts typed commands, arguments, and pipelines for direct interaction with operating system utilities. Mechanically, it reads typed commands, parses options and arguments, then executes programs within the current shell environment. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
