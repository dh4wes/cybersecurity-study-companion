# CPU, RAM, and Storage Drive in context

## 1. Today’s concept in plain language
Today covers CPU, RAM, and Storage Drive within a+ core 1: hardware, boot process, and internal components.

## 2. Mini lecture
The week focus is A+ Core 1: hardware, boot process, and internal components, and today's objective is kick off the week's topic and build the right vocabulary. The main terms in scope are CPU, RAM, and Storage Drive. The focus is on what each one does, how it operates, and how it affects the rest of the system.

CPU is central Processing Unit, executes instructions and coordinates processing tasks. It fetches, decodes, and executes instructions while coordinating arithmetic, logic, and control operations across the system. Mechanically, it fetches instructions, decodes them, and executes operations while registers and caches keep data close to processing cores. RAM is volatile memory used for active processes and short-term data access. It holds active instructions and data in fast volatile memory for immediate CPU access during execution. Mechanically, it holds active instructions and data in fast volatile cells that the CPU can read or update quickly. Storage Drive is persistent storage device such as HDD or SSD for operating system and files. It persists operating system files and user data by writing blocks to magnetic or solid-state media. Mechanically, it writes data into addressable sectors or cells, then retrieves it later through the controller and filesystem mapping. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
