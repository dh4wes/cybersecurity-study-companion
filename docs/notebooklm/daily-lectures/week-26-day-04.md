The main terms in scope are Hashing and Encryption. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Hashing is one-way transformation of data into a fixed-length digest within authentication and access control security workflows. It processes input through a deterministic algorithm to produce a fixed digest for integrity checks and password verification. Mechanically, it runs input through a one-way algorithm that deterministically produces a fixed-length digest, exposing changes when inputs differ. Encryption is reversible transformation that protects confidentiality with keys within authentication and access control security workflows. It uses algorithms and keys to transform plaintext into ciphertext that authorized parties can later decrypt. Mechanically, it transforms plaintext with a key and algorithm into ciphertext, and only the correct key reverses that transformation. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
