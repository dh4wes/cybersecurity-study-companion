The main terms in scope are Timeline Analysis and Post-Incident Review. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Timeline Analysis is reconstruction of incident events in chronological order. It orders events by timestamp to reconstruct attacker actions, response steps, and system changes during an incident. Mechanically, it orders events by timestamp, reconciles clock differences, and fills gaps so cause and sequence become clear. Post-Incident Review is structured retrospective focused on lessons and control improvements. It evaluates timeline, controls, decisions, and communication after closure to improve future response effectiveness. Mechanically, it reconstructs the event, compares actions to expected procedure, and converts gaps into specific follow-up improvements. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
