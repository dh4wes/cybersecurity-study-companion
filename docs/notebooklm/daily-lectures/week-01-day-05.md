# Boot Process and CPU Cache in context

## 2. Mini lecture
The week focus is A+ Core 1: hardware, boot process, and internal components, and today's objective is synthesize the week and create the deliverable. The main terms in scope are Boot Process and CPU Cache. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Boot Process is sequence that moves a system from power-on to a usable operating system session. It transitions from firmware execution to loader, kernel initialization, service startup, and interactive user access. Mechanically, it starts with firmware, loads boot code, initializes the kernel, and then launches system services and user space. CPU Cache is small, high-speed memory on or near the processor that stores frequently used instructions and data. It reduces average memory access time by serving repeated reads faster than system RAM can. Mechanically, it keeps recently or frequently used data close to the core, so many requests finish before the CPU waits on RAM. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
