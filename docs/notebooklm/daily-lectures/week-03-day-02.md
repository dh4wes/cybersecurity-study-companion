# Partition and Boot Media in context

## 1. Today’s concept in plain language
Today covers Partition and Boot Media within a+ core 2: operating systems, installs, file systems, and admin basics.

## 2. Mini lecture
The week focus is A+ Core 2: operating systems, installs, file systems, and admin basics, and today's objective is deepen concepts and connect them to the bigger system. The main terms in scope are Partition and Boot Media. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Partition is logical division of a physical disk that the operating system can format and manage separately. It separates storage space for operating systems, data, recovery areas, or different filesystems. Mechanically, it uses a partition table to mark disk regions, then the operating system formats and mounts each region for use. Boot Media is removable or attached storage prepared to start a system or launch an installer. It lets a machine load setup tools or an operating system when the normal internal drive is unavailable or being replaced. Mechanically, it firmware detects the bootable medium, loads its boot code, and starts the installer or recovery environment. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.

Learn installation methods, boot media, partitions, upgrade paths, and basic deployment concepts. This day is meant to show the topic in action rather than leave it at the definition level.
