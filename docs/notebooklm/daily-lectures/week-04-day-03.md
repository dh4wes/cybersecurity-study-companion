# Ticketing and Documentation in context

## 2. Mini lecture
The week focus is A+ Core 2: security, support professionalism, and troubleshooting method, and today's objective is apply the concepts with examples, drills, or comparisons. The main terms in scope are Ticketing and Documentation. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Ticketing is systematic tracking of support work through numbered cases, status changes, and recorded actions. It organizes requests so issues can be prioritized, handed off, and audited consistently. Mechanically, it creates a case record, updates status as work progresses, and preserves notes so others can continue or review the work. Documentation is written record of configurations, procedures, findings, or changes that explains what was done and why. It preserves knowledge so support and security work can be repeated, handed off, and audited. Mechanically, it captures relevant facts, decisions, and steps in a shared record so future work starts from known context instead of memory. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
