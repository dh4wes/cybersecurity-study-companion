If you can explain those relationships cleanly, the week's deliverable becomes more than homework: it becomes evidence that you understand the system well enough to describe it under exam pressure or in an operations conversation.

Ticketing means systematic tracking of support work through numbered cases, status changes, and recorded actions. Its practical job is that it organizes requests so issues can be prioritized, handed off, and audited consistently. Mechanically, it creates a case record, updates status as work progresses, and preserves notes so others can continue or review the work. Documentation means written record of configurations, procedures, findings, or changes that explains what was done and why. Its practical job is that it preserves knowledge so support and security work can be repeated, handed off, and audited. Mechanically, it captures relevant facts, decisions, and steps in a shared record so future work starts from known context instead of memory. Notice the pattern: the exam target here is not memorizing isolated labels, but recognizing which part of the system is responsible for speed, persistence, coordination, communication, or control.
