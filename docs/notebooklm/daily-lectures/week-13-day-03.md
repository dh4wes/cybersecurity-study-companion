The main terms in scope are Control and Security Framework. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Control is safeguard that prevents, detects, or corrects a security problem. It reduces risk by changing how likely, visible, or damaging an unwanted event can become. Mechanically, it applies a policy, technical setting, or process step that blocks misuse, spots abnormal activity, or limits the resulting damage. Security Framework is structured model that organizes security practices into repeatable domains, functions, or maturity steps. It gives teams a common way to assess coverage, compare gaps, and plan improvement work. Mechanically, it breaks security work into named areas and expected outcomes so organizations can measure current state and target changes. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
