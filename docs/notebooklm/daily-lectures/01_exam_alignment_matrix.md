# Exam Alignment Matrix

## Current canonical note
This matrix follows the live syllabus in the repo as generated on 2026-03-11. The current canonical plan contains 29 active weeks rather than the older 32-week assumption.

| Week range | Primary exam | Secondary exam | Rationale |
| --- | --- | --- | --- |
| Weeks 01-02 | CompTIA A+ Core 1 (220-1201) | None | Hardware, boot process, mobile devices, basic networking, virtualization, and troubleshooting patterns map most directly to A+ Core 1 reasoning. |
| Weeks 03-05 | CompTIA A+ Core 2 (220-1202) | Bridge to CompTIA Network+ (N10-009) and CompTIA Security+ (SY0-701) | Operating systems, filesystems, installs, permissions, procedures, and Linux fundamentals map primarily to Core 2 while supporting later networking and security work. |
| Weeks 06-12 | CompTIA Network+ (N10-009) | None | These weeks teach packet movement, addressing, services, switching, routing, wireless, tools, and troubleshooting, which are the center of the Network+ block. |
| Weeks 13-23 | Google Cybersecurity Professional Certificate | CompTIA Security+ (SY0-701) | These weeks follow the Google certificate sequence and reinforce Security+ style thinking about controls, Linux, SQL, attacks, evidence, detection, and response. |
| Weeks 26-30 | CompTIA Security+ (SY0-701) | None | These weeks are the strongest direct Security+ reinforcement block in the current live syllabus. |
| Week 32 | Cross-certification review (A+ / Network+ / Security+) | Indirect support for all prior tracks | The final buffer week is for consolidation, retrieval, cleanup, and cross-exam review rather than large amounts of new objective coverage. |

## Missing ranges from the older brief
Weeks 24, 25, and 31 are not present in the current canonical data. The lecture pack therefore follows the live repo rather than reconstructing removed weeks from stale instructions.
