## 2. Mini lecture
This is the review day for Google Course 4: Linux and SQL (part 2 - SQL). The goal is to connect the week's main ideas into one working model rather than introduce a new topic. The core terms are SQL Query, SELECT Statement, WHERE Clause, JOIN, Input Validation, and SQL Injection.

Across the week, the key mechanisms were: SQL Query: parses clauses, builds an execution plan, and retrieves or changes rows according to that plan SELECT Statement: specifies columns, tables, filters, and ordering so the database engine can assemble the requested result WHERE Clause: applies boolean conditions to candidate rows so only records that match the filter continue through the query JOIN: matches rows from multiple tables using related keys so combined records can be returned in one result set The value of the review is in seeing how these pieces connect across setup, operation, failure, and troubleshooting.
