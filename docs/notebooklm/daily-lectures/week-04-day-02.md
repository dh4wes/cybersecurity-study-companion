## 2. Mini lecture
The week focus is A+ Core 2: security, support professionalism, and troubleshooting method, and today's objective is deepen concepts and connect them to the bigger system. The main terms in scope are Patch Management and Security Awareness. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Patch Management is process for testing, deploying, and verifying software updates. It identifies missing updates, tests fixes, schedules deployment, and verifies installation across managed assets. Mechanically, it identifies missing fixes, deploys approved updates, and verifies installation so known vulnerabilities are removed systematically. Security Awareness is training and reinforcement that helps users recognize and avoid unsafe behavior. It reduces avoidable mistakes by teaching people how to spot risky requests and follow safe habits. Mechanically, it uses examples, reminders, and repeated practice so users recognize warning signs before acting. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
