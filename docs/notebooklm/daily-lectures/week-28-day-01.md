## 2. Mini lecture
The week focus is CISA incident response materials and playbook thinking, and today's objective is kick off the week's topic and build the right vocabulary. The main terms in scope are Incident Command and Playbook. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Incident Command is leadership structure coordinating decisions and responsibilities during incidents. It assigns command roles, decision authority, and coordination channels so response teams act under a consistent structure. Mechanically, it assigns leadership, roles, and reporting lines so technical response and business decisions stay coordinated under pressure. Playbook is documented response workflow for recurring security scenarios. It documents response steps, roles, decision points, and evidence requirements for a recurring incident scenario. Mechanically, it defines trigger conditions, required steps, and decision points so analysts handle repeat scenarios the same way each time. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
