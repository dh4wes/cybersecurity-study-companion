## 2. Mini lecture
The week focus is A+ Core 2: security, support professionalism, and troubleshooting method, and today's objective is synthesize the week and create the deliverable. The main terms in scope are Knowledge Base and Backup Strategy. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Knowledge Base is shared collection of documented solutions, procedures, and reference notes for recurring issues. It speeds support work by making proven fixes and guidance easy to reuse. Mechanically, it stores structured articles and updates them over time so staff can search, follow, and improve known solutions. Backup Strategy is planned approach for copying and retaining data so it can be restored after loss or corruption. It protects business continuity by ensuring important data can be recovered after failure, deletion, or attack. Mechanically, it chooses what to back up, where to store copies, and how often to test restoration so recovery remains practical. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
