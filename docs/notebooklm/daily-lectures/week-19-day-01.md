## 2. Mini lecture
The week focus is Google Course 5: Assets, threats, and vulnerabilities (part 2), and today's objective is kick off the week's topic and build the right vocabulary. The main terms in scope are Mitigation and Exploit. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Mitigation is action that lowers likelihood or impact of a risk. It selects and implements actions that lower the chance or consequences of a risk event. Mechanically, it reduces either the chance or the consequence of a risk through selected safeguards and process changes. Exploit is code or technique that takes advantage of a vulnerability. It triggers vulnerable behavior through crafted input, code execution, or protocol abuse to gain unauthorized results. Mechanically, it uses crafted input or sequence timing to trigger a weakness and execute behavior the system did not intend to allow. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
