The main terms in scope are Network Protocol and Cloud Computing. The focus is on what each one does, how it operates, and how it affects the rest of the system.

Network Protocol is defined set of rules that controls how devices format, send, receive, and interpret network traffic. It lets different systems communicate predictably by agreeing on message structure and behavior. Mechanically, it specifies message formats, timing, and expected responses so both ends process traffic in the same way. Cloud Computing is on-demand delivery of computing resources over a network instead of relying only on local hardware. It lets organizations scale services and consume shared infrastructure without building every system themselves. Mechanically, it users request shared compute or application resources from a provider, and the provider allocates capacity from pooled infrastructure. Together, these terms describe where work happens, how state changes, and what the visible outcome looks like when the system is operating normally.
